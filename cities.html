When reducing the browser window size, I noticed that the cities row breaks first. This occurs because we specified the breakpoint for the cities row using "col-md," which means that on smaller screens (768px and below), the city names will start stacking vertically. The letter names, specified with "col-sm-2," will continue to display horizontally until the screen becomes even narrower.
